Current problem:

* B is being read as “bee”
* It should sound like “buh”
* S should not say “ess”

Fix the tracing game properly:

Current problem:

* When the child tries to trace, the page scrolls instead
* The letter is just an image
* The tracing does not fill in
* The child is not actually tracing anything

Build the tracing game as a proper interactive tracing activity.

When a child selects a letter:

* Open a centred tracing modal / pop-up box
* The modal must stay fixed on screen
* The background page must not scroll
* Disable body scrolling while tracing modal is open
* Add an X close button
* Show the large letter with dotted guide path
* Show a clear start point
* Show arrows or numbered dots
* Let the child trace using finger, stylus, or mouse
* Use pointer events
* Prevent touch scrolling while tracing
* Use touch-action: none on the tracing canvas
* Fill or highlight the letter path as the child traces
* Show progress percentage
* When enough of the path is traced, mark complete
* Give feedback:
  “Great tracing!”
  “Keep following the dots!”

The child should not be able to drag the page while tracing.

Technical requirements:

* Use SVG or canvas tracing paths
* Track pointer movement
* Compare the child’s trace to the guide path
* Fill in dots/segments as they trace over them
* Require a reasonable completion threshold, for example 75–85%
* Save tracing result to progress and review history

The tracing box should not move or disappear unless:

* The child presses X
* The tracing is complete
* Adult/child presses Back

Fix Fill The Letter / Build The Word game:

Current problem:
The child cannot select a wrong letter. The game only allows the correct letters, which means they cannot make mistakes and the score is always correct.

Fix:

* Allow the child to tap any option, including wrong letters
* If the child chooses the wrong letter, accept the answer as wrong
* Show feedback:
  “Good try! The correct letter was ___.”
* Move to the next question after feedback
* Save it as incorrect in review history

When the child taps any letter option:

* Speak the phonics sound of that letter
* Do not speak the alphabet name unless the adult chooses letter-name mode
* Example:
  Child taps “s”
  Voice says “sss”
  not “ess”

For word building:

* The child should be able to build the word using choices
* If they build it wrong, mark incorrect
* Show the correct word
* Move to the next question

Review history must save:

* Word shown
* Missing letter
* Options shown
* Child’s chosen letter
* Correct letter
* Whether correct
* Explanation

Example:
Word:
c _ t

Child chose:
o

Correct answer:
a

Result:
Incorrect

Explanation:
The word is cat. The missing sound is a.
